Crie o seu Portfólio para Data Science

Crie e hospede gratuitamente o seu portfólio/blog de Ciência de Dados utilizando o HTML UP e o GitHub Pages.


Quando você está iniciando a sua carreira em Ciência de Dados é muito importante que você demonstre os seus conhecimentos e habilidades através de um portfólio de projetos. (especialmente se você vem de uma área completamente diferente).
Através de um bom portfólio de projetos é possível que você apresente suas experiências com projetos, a sua linha de raciocínio para resolução de problemas, organização e muitas outras qualidades. Embora, durante a construção de um portfólio é necessário que você se dedique em construir a melhor versão de si, que mostre aos recrutadores que está realmente preparado para o mercado de trabalho.

Pensando nisso, neste artigo eu quero trazer uma solução para quem pretende construir seu portfólio, mas que não quer em gastar com hospedagem, criação, design, etc.
Gostaria de primeiramente dar os créditos a esse tutorial ao devido autor Joshua Fluke, que através desse vídeo disponibiliza um passo a passo de como você pode construir um portfólio pessoal. Gostei muito da ideia do conteúdo e pensei em criar o meu blog/portfólio pessoal para compartilhar artigos e projetos.

Let´s go!

O primeiro passo é acessar o site HTML UP.


Este site disponibiliza diversos templates para websites responsivos criados em HTML, com a estilização em CSS e todas as funcionalidades em Javascript (Sim, se você quiser fazer algumas alterações terá que aprender um pouquinho dessas linguagens, sorry!)
Você pode ver uma pré-visualização antes de escolher o template, basta clicar em live demo que você será redirecionado para outra página onde mostra todas as funcionalidades do template selecionado.


Após escolher o template que mais combina com você, basta clicar em fazer download. (uma dica é criar uma pasta para armazenar todos os arquivos)
Feito o download, será necessário abrir estes arquivos em um editor de texto para que possamos fazer as alterações no site e para testar se tudo está funcionando corretamente.
No meu caso irei utilizar o VS Code, pois é o editor que utilizo e que disponibiliza diversas extensões que facilitam muito na hora de codar.

Abrindo arquivos:

Para abrir os arquivos corretamente, é preciso selecionar a pasta inteira, pois assim podemos trabalhar com todos os arquivos que correspondem as páginas do site.


Depois que abrir a pasta, é muito provável que aparece arquivos semelhantes a esses:


Pelos testes que fiz utilizando outros templates, o arquivo index.html será sempre a página inicial do site e as demais serão as outras abas. Na pasta assets você encontra o arquivo CSS onde está toda a estilização que foi utilizada. A pasta images contém as imagens que estão sendo usadas no site, onde futuramente você poderá armazenar as imagens que deseja utilizar nas páginas.
Uma boa prática que aprendi com os desenvolvedores web é visualizar as modificações que estão sendo feitas na página em tempo real. Para fazer isso, o VS Code disponibiliza uma extensão chamada Live Server.
Infelizmente essa extensão não vem instalada junto com o VS Code. Podemos instalá-la clicando na aba "extensões" e buscando por Live Server.


Agora que já estamos familiarizados com os arquivos e já adicionamos o Live Server para o nosso arsenal de extensões, iremos testar se o site realmente está funcionando e também se as modificações estão sendo feitas em tempo real.
Para testar o Live Server nós podemos clicar em Go Live no canto direito da tela:


Feito isso, se tudo der certo é para abrir uma página no seu navegador mostrando as alterações feitas e todas as outras funcionalidades do site.

Hospedando o site no GitHub Pages

Muitos não conhecem essa funcionalidade, mas o GitHub permite que seja hospedado site estáticos e algumas aplicações simples por meio do GitHub Pages.
O primeiro passo é criar um repositório para a nossa aplicação:


O nome desse repositório deve ser o seu user do GitHub + github.io, como por exemplo o meu: brayannmb.github.io
O repositório deve ser público e não é necessário adicionar um README.
Após criar o repositório, iremos adicionar os arquivos baixados do template escolhido. Um ponto importante aqui é adicionar todos os arquivos de uma só vez.
O segundo passo é permitir que o GitHub utilize este repositório como uma aplicação web e coloque o site para funcionar. Para fazer isso, devemos selecionar o repositório e clicar em configurações:


O terceiro passo é buscar por GitHub Pages na tela de configurações e clicar em "Confira aqui!":


Feito isso, seremos redirecionados para uma outra página onde mostra todas as configurações que podem ser feitas no GitHub Pages. Para que o site seja hospedado corretamente e disponível para ser acessado, devemos mudar a origem para a branch principal:


Após realizar esses passos o seu site já estará pronto para uso, basta acessar o link disponibilizado nessa mesma página onde foi feito a alteração da branch:


Prontinho! Site criado, hospedado e disponível para você fazer as suas alterações da maneira que achar mais adequada.
